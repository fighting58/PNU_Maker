# 📖 makePNU Workbench 사용 가이드

**makePNU Workbench**는 엑셀 또는 CSV 데이터 내의 지번 주소를 기반으로 19자리 **PNU(필지식별번호)**를 자동 생성해 주는 전문 대시보드 도구입니다.

---

## 1. 입력 파일 형식 (XLSX, CSV)

### 지원 파일 확장자
-   `.xlsx`, `.xls` (Microsoft Excel)
-   `.csv` (Comma Separated Values)

### 필수 데이터 항목
PNU를 생성하기 위해서는 반드시 다음 두 가지 성격의 데이터 열(Column)이 포함되어 있어야 합니다.
1.  **행정구역명**: 법정동/리 명칭이 포함된 열 (예: `강원도 홍천군 두촌면 괘석리`, `수원시 인계동`)
2.  **지번**: 필지의 본번과 부번이 포함된 열 (예: `723`, `산 123-4`, `45-1`)

> [!TIP]
> **데이터 프리뷰**: 파일을 업로드하면 원본 파일의 **모든 열(Columns)**이 프리뷰 테이블에 즉시 표시되어 전체적인 데이터 구조를 바로 확인할 수 있습니다.

---

## 2. 단계별 상세 설명

### [Step 1] 데이터 원본 업로드
-   사이드바의 **'데이터 원본 업로드'** 영역에 파일을 업로드합니다.
-   **자동 감지(Quick-Sync)**: 업로드한 파일에 이미 `PNU` 또는 `pnu`라는 이름의 컬럼이 있다면, 별도의 생성 실행 없이도 하단의 **'PNU 리스트 출력'** 영역이 즉시 활성화됩니다.
-   **복사** 버튼을 눌러 클립보드에 저장할 수 있습니다.

### [Step 2] 컬럼 매핑 및 실행
-   **행정구역 (동/리)**와 **지번 (본번-부번)**에 해당하는 컬럼을 선택합니다. (시스템이 자동으로 추천 항목을 선택해 둡니다.)
-   설정이 완료되면 **[PNU 생성 실행]** 버튼을 클릭합니다.

### [Step 3] PNU 생성 및 검증 (자동)
-   **지능형 매칭**: `강원도`를 `강원특별자치도`로 인식하는 등 최신 행정구역 명칭 변화를 자동으로 감지하여 매칭 정확도를 극대화합니다.
-   **실시간 처리**: 브라우저 로컬 캐싱을 통해 대량의 데이터도 수 초 내에 분석합니다.

### [Step 4] 결과 확인 및 PNU 리스트 추출
-   **결과 프리뷰**: 생성된 19자리 PNU와 오류 메시지(`PNU_ERROR`)를 테이블에서 확인합니다.
-   **PNU 리스트 검증**: 하단 리스트 영역에는 **정확히 숫자로 된 19자리** 데이터만 추출됩니다.
    -   **정상/누락 통계**: 전체 입력 건수 중 리스트로 성공적으로 추출된 건수와 누락된 건수(형식 오류 등)를 실시간으로 비교하여 보여줍니다.
-   **데이터 저장**: 우측 상단의 **[결과 저장]** 버튼을 통해 최종 데이터를 엑셀로 내려받을 수 있습니다.
-   **새 작업 시작**: 모든 작업이 끝난 후 상단 헤더의 **[새 작업]** 버튼을 클릭하면, 현재 리스트와 상태가 초기화되어 다시 [Step 1] 상태로 돌아갑니다.

---

## 3. DB 관리 및 도구 제어

사이드바 하단의 **[관리 도구 열기]**를 클릭하여 상세 설정을 관리할 수 있습니다.

1.  **최신 DB 동기화**: [**행정표준코드관리시스템**](https://www.code.go.kr/) 서버에서 최신 법정동 코드를 다운로드합니다.
    - 현재 프로젝트 폴더에 **official_bjd.zip** 파일로 저장됩니다.
2.  **수동 업로드**: 오프라인 환경을 위해 ZIP 파일 형태의 직접 업로드를 지원합니다.
3.  **패널 제어**: 관리 도구 창 우측 상단의 화살표 버튼(▲, ▼)을 클릭하여 창을 접거나 펼칠 수 있습니다.

## 4. 문제 해결 (Troubleshooting)

### Q. 강원도나 전라도 주소가 매칭되지 않습니다.
-   충청남도(충남), 충청북도(충북), 전라남도(전남), 전라북도(전북), 경상남도(경남), 경상북도(경북), 강원특별자치도(강원) 등 약칭은 자동으로 변환되나 오류를 방지하기 위해 **정식 시도 명칭**을 사용하시길 추천합니다.
-   행정구역의 일부만 입력할 경우 가장 먼저 매칭이 된 행정구역을 사용합니다. 오류 방지를 위해 **시군구-읍면동리** 단위 이상의 행정구역명을 입력하시길 권장합니다.
-   정확한 행정구역을 입력했음에도 매칭이 되지 않는 경우, **[최신 DB 동기화]**를 통해 최신 정보를 반영해 주세요.

### Q. PNU 리스트의 숫자가 전체 데이터 수보다 적습니다.
-   시스템은 리스트 추출 시 **엄격한 검증(19자리 숫자)**을 거칩니다. 주소가 부정확하거나 지번 형식이 잘못되어 PNU가 생성되지 않은 항목은 리스트에서 제외되며, 상단의 '오류/누락' 통계에 집계됩니다.
-   입력 데이터(xlsx, csv)에서 오류가 발생한 행의 행정구역명과 지번을 수정한 후 재시도해 보세요.
